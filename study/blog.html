# 2. npm包同步问题

由于插件的包是发布在npm仓库上的，但是我们通常使用的都是cnpm的源，因此在更新不及时的情况下，需要我们手动去同步仓库。

仓库同步链接 (opens new window)

# 3. lx-crud插件

模板插件是发布在开源仓库的，lx-crud插件时发布在内部环境的，目前暂时没找到如何将模板发布到内部仓库仍然可以使用的方法。暂时的处理方法为：

  1. vue add 模板来引入基础配置
  2. 切换npm源到本地仓库
  3. 使用npm重新安装内部插件
